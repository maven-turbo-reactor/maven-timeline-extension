See how your Maven build actually runs.

A drop-in Maven extension that records every goal across every worker thread and renders an interactive, single-file HTML timeline — with CPU, heap, live threads and resolver I/O charts all aligned to the same zoomable time axis.

What you get

One HTML file, opened in any browser, that turns an opaque build log into a profile you can read.

🧵

Worker-thread swimlanes

A Gantt view with one lane per Maven worker thread, so you can instantly see parallelism, stalls and the critical path through your reactor.

📊

Aligned resource charts

CPU utilization (system & process), heap usage with GC markers, live thread count and resolver download/upload throughput — all sharing the timeline's x-axis.

🔍

Zoom & pan, everything synced

Scroll, pinch or double-click to zoom; drag or two-finger swipe to pan. A navigator strip acts as a scrollbar. Every chart and crosshair stays locked to the same window.

⏱️

The <prepare> phase

A synthetic, hatched block surfaces the gap between a module's init and its first real goal — usually dependency resolution — so hidden wait time becomes visible.

🎨

Goals colored by type

compile, test-compile, test and deploy each get a distinct hue, with a per-module saturation tweak so goals of the same module read together. Hover any bar for full detail.

📦

Self-contained, zero install

The report inlines its data and the D3.js chart code into a single static file — commit it, email it, or attach it to a CI artifact. No server required.

Get started in one step

Register the extension; the report is written on every build.

  1. Add the extension to .mvn/extensions.xml in the root of your project: 
    <extensions>
    <extension>
        <!-- https://github.com/maven-turbo-reactor/maven-timeline-extension -->
        <groupId>com.github.seregamorph</groupId>
        <artifactId>maven-timeline-extension</artifactId>
        <version>0.1</version>
    </extension>
</extensions>
  2. Build as usual — run any Maven command, e.g. mvn verify.
  3. Open the report — find it at target/timeline/build-report.html under your root project directory and open it in a browser.

Works with both Maven 3 and Maven 4. The chart UI is built with D3.js.

How to read the timeline

A quick tour of the report you just opened.

Execution lane

Each bar is a single Maven goal placed on its worker thread. Width = duration. Hover to see the module, goal, worker, start time and elapsed time.

Navigator

The faint strip below the Gantt shows active-task count across the whole build. Drag a window or resize its handles to zoom every chart at once.

Crosshair

Move the cursor over any chart and a shared red crosshair appears across all of them, so you can correlate a CPU spike or GC pause with the goal that caused it.
